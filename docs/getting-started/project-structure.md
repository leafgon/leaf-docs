# Project Structure

## Overview
At authoring time, LEAF projects are organized around graph pages and named resources. Naming is a first step: your nickname and app name form a URI-like identity.

Typical structure in this docs repo:
- Concept docs in `docs/core-concepts/`
- Architecture docs in `docs/architecture/`
- Practical guides in `docs/getting-started/` and `docs/examples/`

## When to use
Use this page when deciding where to place new docs or how to model LEAF app identity.

## Example
Reference another page-level spell from a different page using a `leafgraph` abstraction node.
