# Design Goals

## Overview
The source corpus emphasizes these goals for LEAF:
- Accessibility: enable non-professional developers to author useful software.
- Lower cognitive load: represent flow explicitly as graphs instead of hidden control flow.
- Practical expressiveness: support abstraction, iteration, and data handling in one model.
- Human-centered programming: prioritize understandable composition and refactoring.

## When to use
Use this page when evaluating whether a LEAF design direction aligns with the language philosophy.

## Example
A design choice like using an `anchor` node to deactivate a subgraph aligns with readability and safe iteration during development.
